What is a Form

An HTML Form tag is an 'encapsulating' tag with other tags inside it that define their purpose.
The HTML <form> element is used to create an HTML form for user input (below is a simple form).





A form tag in editor will be formatted with a label that has the 'for' attribute assigned the same text
as the <input> tag id and name attribute for look up when sending a form from our page.
<form>
<label for="fname">First Name:</label><br>
<input type="text" id="fname" name="fname"><br>
<label for="lname">Last Name:</label><br>
<input type="text" id="lname" name="lname"><br>
<input type="submit">
</form>
 This form currently does nothing of interest as it doesn't even respond to your input. One main part of
forms is the backend interaction it has with scripts; in this case we would likely add your information
to a database for storage. We don't need to do anything that complicated as of yet.

Below is the same form, however under the hood things have changed. (try it)




Now you can see the data we sent out with the form as I directed it to a traceback script hosted by
http://jkorpela.fi/ and we get back a table with the id attributes and the values we gave the form.
It is typical of webpages not to show us our own conformation when making form submissions but this is
meant to show you more of what we learned in 118 this week. The following is our new code for the form tag is
<form action="http://jkorpela.fi/cgi-bin/echo.cgi">
<label for="fname">First Name:</label><br>
<input type="text" id="fname" name="fname"><br>
<label for="lname">Last Name:</label><br>
<input type="text" id="lname" name="lname"><br>
<input type="submit" value="Send">
</form>
 Looking here we changed two major components for this to function the way it has in the initial form tag
<form action="http://jkorpela.fi/cgi-bin/echo.cgi"> as the action attribute specifies where to send the form-data when a form is submitted
